# Genesis Player - Capacitor Mobile App Setup

This project is now configured to run as both a web application and a native Android/iOS app using Capacitor.

## Prerequisites

- Node.js (v24.x or higher)
- Android Studio (for Android development)
- Xcode (for iOS development, macOS only)

## Project Structure

```
Music-Player-Apk/
├── android/              # Native Android project (generated by Capacitor)
├── backend/              # Express.js backend server
├── public/               # Web app files (Capacitor webDir)
│   ├── js/              # JavaScript modules
│   ├── css/             # Stylesheets
│   ├── assets/          # Images, fonts, etc.
│   └── index.html       # Main HTML file
├── capacitor.config.json # Capacitor configuration
└── package.json
```

## Installation

1. Install dependencies:
```bash
npm install
```

## Running the Web App (Development)

To run the app in a browser with the backend server:

```bash
npm start
```

Then open http://localhost:1552 in your browser.

## Building for Android

### Step 1: Configure Backend URL

Before building for mobile, you need to deploy your backend and update the API configuration:

1. Deploy your backend to a hosting service (e.g., Vercel, Heroku, Railway)
2. Update `public/js/config.js` with your production backend URL:

```javascript
export const API_CONFIG = {
    baseURL: '',
    productionURL: 'https://your-backend-url.com', // Add your deployed backend URL here
    // ...
};
```

### Step 2: Sync Web Assets to Android

After making changes to your web app, sync them to the Android project:

```bash
npm run cap:sync
```

Or manually:
```bash
npx cap sync android
```

### Step 3: Open in Android Studio

```bash
npm run cap:open:android
```

Or manually:
```bash
npx cap open android
```

### Step 4: Build and Run

1. In Android Studio, wait for Gradle to finish syncing
2. Connect an Android device or start an emulator
3. Click the "Run" button (green play icon) or press Shift+F10

## Building for iOS (macOS only)

### Step 1: Add iOS Platform

```bash
npx cap add ios
```

### Step 2: Sync Assets

```bash
npx cap sync ios
```

### Step 3: Open in Xcode

```bash
npx cap open ios
```

### Step 4: Build and Run

1. In Xcode, select your target device/simulator
2. Click the "Run" button or press Cmd+R

## Capacitor Commands Reference

- `npx cap sync` - Copy web assets and sync native plugins
- `npx cap sync android` - Sync only Android
- `npx cap sync ios` - Sync only iOS
- `npx cap open android` - Open Android project in Android Studio
- `npx cap open ios` - Open iOS project in Xcode
- `npx cap add android` - Add Android platform
- `npx cap add ios` - Add iOS platform

- **@capacitor/android** - Android platform
- **@capacitor/ios** - iOS platform
- **@capacitor/app** - App lifecycle and info

- **@capacitor/splash-screen** - Splash screen control
- **@capacitor/status-bar** - Status bar styling

## Important Notes

### API Calls

The app uses a custom API configuration (`public/js/config.js`) that automatically detects whether it's running in a browser or as a mobile app:

- **Browser**: API calls go to the same origin (localhost:1552)
- **Mobile App**: API calls go to the configured `productionURL`

### Local Files

When running as a mobile app, the file picker will use native Android/iOS file pickers. The app stores music metadata in IndexedDB (Dexie) which works on both web and mobile.

### Debugging

- **Android**: Use Chrome DevTools by navigating to `chrome://inspect` while the app is running
- **iOS**: Use Safari Web Inspector (Safari > Develop > [Your Device])

## Troubleshooting

### "Module not found" errors in Android

Run `npx cap sync android` to ensure all web assets are copied to the Android project.

### API calls failing in mobile app

Make sure you've set the `productionURL` in `public/js/config.js` and that your backend is deployed and accessible.

### Gradle build errors

1. Open Android Studio
2. File > Invalidate Caches / Restart
3. Clean and rebuild the project

### White screen on app launch

Check the browser console in Chrome DevTools (`chrome://inspect`) for JavaScript errors.

## Building for Production

### Android APK/AAB

2. Follow the wizard to create a keystore and sign your app
3. Choose "release" build variant

### iOS App Store

1. In Xcode: Product > Archive
2. Follow the wizard to upload to App Store Connect
